Instructions de compilation et d'installation

Le répertoire netbeans et tous ses sous-répertoires sont conçus pour être compilés par l'IDE NetBeans. Chaque sous-répertoire (notamment Sicade) est un module conçu pour la plateforme NetBeans. Cette dernière est requise sur le poste du client pour l'exécution de l'application. Notez que la plateforme NetBeans n'est pas l'IDE complet, mais seulement une infrastructure de base pour le développement d'applications.

Les répertoires versionnés sur SVN n'incluent pas le sous-répertoire private, qui contient des chemins propres à l'installation des projets de l'IDE NetBeans sur un poste local.

L'application Sicade nécessite les extensions suivantes, qui doivent être installées sur le poste du client. Nous n'incluons pas ces extensions dans les bibliothèques livrées avec l'application afin d'éviter d'alourdir les mises à jours, et aussi afin de réduire les doublons (ces extensions étant quand même partagées par plus d'une application):

  • Java Advanced Imaging
  • Java3D
  • Java look and feel Graphics Repository
  • Pilote JDBC de PostgreSQL

L'application peut être distributée sous forme de fichier ZIP en sélectionnant Build ZIP distribution dans le menu contextuel du projet. Dans ce fichier ZIP, les modifications suivantes doivent être effectuées manuellement:

  • Remplacer le fichier etc/sicade.conf par celui qui apparaît ici dans ce répertoire.
