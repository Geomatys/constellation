Observations environnementales provenant d'une base de données d'images.

Les données sont stockées comme des images ordinaires au format PNG, RAW, ASCII, JPEG-2000, etc. (le format JPEG classique est à éviter). Une base de données (par exemple PostgreSQL) est utilisée pour stocker des méta-données (enveloppe géographique, signification des valeurs des pixels, etc.). Cette base de données est aussi utilisée comme un index pour effectuer des recherches rapides. La base de données, ainsi que les fichiers des images, sont habituellement situés sur un serveur distant. La connexion à la base de données se fait via JDBC, qui peut fonctionner à travers un réseau. La récupération des valeurs de pixels des images peuvent se faire de deux façons:

  • Les images sont récupérées par FTP et les traitements effectués sur le poste du client. C'est l'approche qui demande le moins de temps de processeur sur le serveur, mais peut exiger une bande passante élevée (chaque images fait plusieurs méga-octets, et un traitement nécessite souvent plusieurs images).

  • Le client se connecte au serveur via les RMI (Remote Method Invocation). Cette approche ajoute une charge de travail du côté du serveur, mais nécessite peu de bande passante si seulement quelques pixels sont demandés (par exemple seulement les valeurs aux positions de pêches). Notez que cette approche devient systématiquement moins efficace que l'approche par FTP si l'utilisateur demande un nombre de valeurs d'une même image qui représente une proportion significative du nombre de pixels que contient cette image (vaut mieux alors transférer la totalité de l'image chez le client).
