Implémentations des interfaces du paquet {@link net.sicade.observation observation}. Les classes définies dans ce paquet fournissent assez de fonctionalités pour un service de base utilisant des requêtes SQL pour se connecter à une base de données. Ces classes peuvent être étendues dans des paquets spécialisés, par exemple {@link net.sicade.observation.fishery.sql fishery}. Les classes définies dans ce paquet se répartissent en deux branches:

  • {@link net.sicade.observation.sql.Table} est la racine des classes qui représenteront une connexion SQL (Structured Language Query) à une base de données. Toutes les tables sont conçues pour accéder aux données d'un serveur distant, la plupart du temps en envoyant simplement la requête SQL sur le réseau. Toutefois certaines tables préviligieront plutôt une utilisation via les RMI (Remote Method Invocation). Un exemple de table accédée à distance via RMI est {@link net.sicade.observation.coverage.sql.GridCoverageTable}, car elle peut manipuler des données volumineuses que l'on ne souhaite pas forcément transférer chez le client.

  • {@link net.sicade.observation.sql.Entry} est la racine des implémentations des interfaces définies dans le paquet {@link net.sicade.observation observation} et les sous-paquets spécialisés. Il s'agira souvent (mais pas obligatoirement) d'un enregistrement dans une {@linkplain net.sicade.observation.sql.Table table}, d'où le nom {@code Entry}.

Les {@linkplain net.sicade.observation.sql.Table tables} sont liées de près aux bases de données sous-jacentes et sont internes aux implémentations; les utilisateurs des interfaces du paquet {@linkplain net.sicade.observation observation} ne devraient pas les voir. Ces tables seront référencées dans les objets {@link net.sicade.observation.sql.Entry}, par exemple pour obtenir l'ensemble des {@linkplain net.sicade.observation.sql.ObservationEntry observations} à une {@linkplain net.sicade.observation.sql.StationEntry station}.
