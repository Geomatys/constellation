Interfaces communes à tous types d'observations supportées par Sicade. Le type le plus couramment utilisé est la {@linkplain net.sicade.observation.Measurement mesure}, qui est un cas particulier d'{@linkplain net.sicade.observation.Observation observation} où la valeur observée est un scalaire. Chaque observation est associée (directement ou indirectement) à un {@linkplain net.sicade.observation.Phenomenon phénomène} mesuré à l'aide d'une certaine {@linkplain net.sicade.observation.Procedure procédure} à une certaine {@linkplain net.sicade.observation.Station station}.

Les notions de phénomène, procédure et de station peuvent être spécialisés en fonction de la discipline étudiée. Ces spécialisations apparaissent dans des sous-paquets. Par exemple dans le cas des images (sous-paquet {@link net.sicade.observation.coverage coverage}), une procédure est une {@linkplain net.sicade.observation.coverage.Operation opération} appliquée sur les images. De même dans le cas des pêches palangrières (sous paquet {@link net.sicade.observation.fishery fishery}), une station est une {@linkplain net.sicade.observation.fishery.LongLine ligne de palangre} ou un {@linkplain net.sicade.observation.fishery.SeineSet coup de senne}.

Le point de départ le plus pratique pour utiliser ce paquet est {@link net.sicade.observation.Observations#getDefault()}.
